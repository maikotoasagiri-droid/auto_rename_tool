# 技術仕様書：自動リネームプログラム

## 1. 概要
Windowsのダウンロードフォルダを監視し、新規作成されたファイルを特定の命名規則に従って自動的にリネームするPythonスクリプト。

## 2. リネーム規則
- 形式: `YYYYMMDD_元のファイル名_v.拡張子`
- 例: `test.pdf` → `20240508_test_v.pdf`
- 重複回避: 既に同名のファイルが存在する場合、末尾に `_v2`, `_v3` とカウントアップする（検討中）。

## 3. 技術スタック
- **言語**: Python 3.x
- **監視ライブラリ**: `watchdog` (Windows APIを効率的に利用可能)
- **設定管理**: `config.json` (監視パスや除外リストの設定)
- **実行形態**: バックグラウンドプロセス（`pythonw.exe` または Windowsサービス化）

## 4. 処理フロー
1. スクリプト起動時に `config.json` を読み込む。
2. `watchdog` を使用して指定フォルダの `created` イベントを監視。
3. ファイルが作成されたら、一定時間待機（書き込み完了を待つ）。
4. ファイル名を取得し、日付プレフィックスと `_v` を付加した新名称を生成。
5. `os.rename` でリネーム実行。
6. 実行ログを `rename_log.txt` に記録。

## 5. 考慮事項
- **書き込みロック**: ブラウザがダウンロード中のファイルはロックされているため、リネーム前にリトライ処理を入れる必要がある。
- **一時ファイル**: `.tmp` や `.crdownload` (Chrome) などの一時ファイルは無視し、最終的な確定時（リネームイベント等）にトリガーさせる。
